# v0.2.0-rc1 Release Notes

**Date:** 2026-05-24
**Status:** Release Candidate 1

## Summary

v0.2.0-rc1 is the first release candidate of the v0.2 feature-complete release.
All 12 implementation stages (20a through 27c) are complete. No new pipeline
features are added in this rc — the focus is release hardening and documentation
consistency.

## Major Additions (since v0.1.0-beta)

- **FE/ppois BigWig conversion** — MACS3 signal track bedGraph files are now
  automatically converted to browser-friendly BigWig format when
  `genome_resources.<genome>.chrom_sizes` is configured.
- **QC summary Python refactor** — per-sample and project-level QC summary
  assembly migrated from shell `printf`/`tail`/`cut` to header-based Python
  scripts (`assemble_qc_summary.py`, `aggregate_qc_summary.py`).
- **Minimal result manifest** — project-level `result_manifest.tsv` records
  core output existence with a 10-column TSV schema.
- **Target builder cleanup** — `rule all` block refactored into 9 helper
  functions with zero DAG behavior change.
- **Assay policy and IDR contract documentation** — behavioral contracts for
  ChIP-seq, CUT&Tag, and ATAC-seq documented from actual code.
- **Public data validation plan** — candidate datasets, acceptance criteria,
  and execution tiers for reproducible validation.
- **CI/CD expansion** — fast PR checks expanded to validation plus 9 Python
  test suites (10 commands). Public data execution remains manual/external.

## Behavior Changes

None. All existing output paths, rule semantics, config schema, and DAG
behavior are preserved. The `rule all` target builder was refactored into
helpers but the resulting DAG is byte-identical.

## Known Limitations (v0.2.0-rc1)

- **Public data validation** — datasets are selected and documented but have
  not been executed. Manual downsampled/full runs are planned but external to CI.
- **GC bias metrics** — Picard CollectGcBiasMetrics is not yet implemented.
- **3+ replicate IDR** — IDR supports exactly 2 biological replicates.
- **CUT&Tag IDR** — not supported in v0.2.
- **ATAC IDR** — not supported in v0.2.
- **ATAC broad peaks** — not supported in v0.2.
- **No bundled public datasets** — reference data must be prepared externally.
- **No container images** — Conda environments are the truth source.

## Manual Public Data Validation Status

Four candidate datasets are selected (ENCSR000DYI, ENCSR000AKB, ENCSR254KDA,
GSE145187) covering TF ChIP, broad histone, ATAC, and CUT&Tag. The
`scripts/prepare_public_validation_inputs.py` helper prints the dataset
inventory and metadata checklist. No data has been downloaded.

## Upgrade Notes from v0.1.0-beta

- **Environment:** No changes to runner or tool environments. Existing
  `chipseq-runner` and rule-specific envs are compatible.
- **Config:** No new required config keys. `genome_resources.<genome>.chrom_sizes`
  is optional — set it to enable FE/ppois BigWig conversion.
- **Output paths:** Unchanged. All existing paths are preserved.
- **QC summary format:** Output TSV is byte-compatible with v0.1.0-beta
  (37 columns, LF line terminators). The assembly method changed from shell
  to Python but the contract is preserved.
- **New outputs:** `results/multiqc/result_manifest.tsv` (10-column TSV).
  FE/ppois BigWig files (`*.FE.bw`, `*.ppois.bw`) when `chrom_sizes` is
  configured.
