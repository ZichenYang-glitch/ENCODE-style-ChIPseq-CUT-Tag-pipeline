# Stage 8c: GitHub Actions CI Wiring — Design Spec

**Date:** 2026-05-18
**Status:** design review
**Scope:** CI workflow for validation, dry-run smoke, and optional real execution
**Excluded:** Conda env splitting, prefix removal, workstation docs (all Stage 8d)

---

## 1. Goals

1. Add a GitHub Actions CI workflow that runs on PRs and pushes to active
   branches, providing fast feedback on config validation, sample-sheet
   validation, and DAG connectivity.
2. Keep PR CI fast and predictable — target ~1-2 minutes with a warm
   environment cache.
3. Provide an optional manual real-execution job (`workflow_dispatch`) that
   runs the Stage 8b tiny real-execution harness.
4. No Conda environment refactoring — Stage 8d handles that.
5. No generated outputs committed. All harness outputs stay under `/tmp`.

---

## 2. Workflow structure

**File:** `.github/workflows/ci.yml`

### 2.1 Triggers

| Event | `fast-checks` | `real-execution` |
|-------|:------------:|:----------------:|
| `pull_request` | runs | skipped |
| `push` to `["main", "stage*"]` | runs | skipped |
| `workflow_dispatch` | runs | runs |

`stage*` covers all Stage branches (8a, 8b, 8c, 8d, etc.) without
needing individual entries.  The glob is quoted to avoid YAML alias
parsing issues.

On `workflow_dispatch`, both jobs execute: the fast baseline checks
validate the environment before the real-execution job starts.

### 2.2 Job: `fast-checks`

```yaml
name: CI

"on":
  pull_request:
  push:
    branches: [main, "stage*"]
  workflow_dispatch:

permissions:
  contents: read

concurrency:
  group: ${{ github.workflow }}-${{ github.ref }}
  cancel-in-progress: true

jobs:
  fast-checks:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Setup micromamba
        uses: mamba-org/setup-micromamba@v3
        with:
          environment-file: workflow/envs/chipseq.yml
          environment-name: chipseq
          cache-environment: true
          cache-environment-key: chipseq-${{ runner.os }}-${{ hashFiles('workflow/envs/chipseq.yml') }}

      - name: Validate default config
        shell: micromamba-shell {0}
        run: python3 scripts/validate_samples.py --config config/config.yaml

      - name: Validation stress tests
        shell: micromamba-shell {0}
        run: python3 test/test_validation_stress.py

      - name: Stage 8a dry-run smoke profiles
        shell: micromamba-shell {0}
        run: python3 test/test_stage8_smoke_profiles.py
```

### 2.3 Job: `real-execution`

```yaml
  real-execution:
    runs-on: ubuntu-latest
    if: github.event_name == 'workflow_dispatch'
    steps:
      - uses: actions/checkout@v4

      - name: Setup micromamba
        uses: mamba-org/setup-micromamba@v3
        with:
          environment-file: workflow/envs/chipseq.yml
          environment-name: chipseq
          cache-environment: true
          cache-environment-key: chipseq-${{ runner.os }}-${{ hashFiles('workflow/envs/chipseq.yml') }}

      - name: Stage 8b tiny real execution
        shell: micromamba-shell {0}
        run: python3 test/test_stage8b_tiny_execution.py
```

### 2.4 Dependency strategy

- **`setup-micromamba@v3`** installs and configures micromamba on the
  runner. Micromamba is NOT preinstalled on GitHub-hosted runners — the
  action handles installation.
- `environment-file: workflow/envs/chipseq.yml` provides the full conda
  environment specification (name: `chipseq`).
- `environment-name: chipseq` makes the generated `micromamba-shell`
  wrapper and cache behavior explicit.
- `cache-environment: true` with a hash key on `chipseq.yml` caches
  the solved environment. Subsequent runs restore in ~30-60 seconds.
  The environment is re-solved only when `chipseq.yml` changes.
- `micromamba-shell {0}` is the shell wrapper generated by
  `setup-micromamba` — it activates the named environment before
  executing the script step.

### 2.5 Exit code handling

- `fast-checks`: any non-zero exit fails the job. None of the three
  commands produce exit code 2.
- `real-execution`: Stage 8b harness returns exit code 2 when required
  tools are not found. In the `workflow_dispatch` context, environment
  unavailability is a real failure — the job fails on exit code 2.

### 2.6 Stage 8b manual-only for now

`real-execution` is gated behind `workflow_dispatch` for the initial
deployment. After the environment cache and runtime prove stable across
several manual runs, it can be promoted to required PR CI by removing
the `if:` gate (or adding a second trigger). This is documented as a
future step, not part of Stage 8c.

## 3. Runtime estimates

### `fast-checks`

| Step | Cold (first solve) | Warm (cache hit) |
|------|-------------------|------------------|
| Checkout | ~5 s | ~5 s |
| Micromamba env solve | ~5-8 min | ~30-60 s |
| validate_samples.py | ~1 s | ~1 s |
| test_validation_stress.py | ~2 s | ~2 s |
| test_stage8_smoke_profiles.py | ~15-30 s | ~15-30 s |
| **Total** | **~6-10 min** | **~1-2 min** |

### `real-execution`

| Step | Cold | Warm |
|------|------|------|
| Micromamba (shared cache) | — | ~30-60 s |
| test_stage8b_tiny_execution.py | — | ~30-45 s |
| **Total** | — | **~1-2 min** |

Both jobs share the same cache key, so `real-execution` always hits
the cache if `fast-checks` has already run.

## 6. Self-review notes

- No Stage 8d env refactor creep — Conda environment is used as-is.
- No reliance on local absolute paths — CI uses `ubuntu-latest` with
  `actions/checkout` and `setup-micromamba`.
- All commands work on GitHub-hosted Ubuntu runners.
- `fast-checks` runs in ~1-2 min warm, ~6-10 min cold.
- `real-execution` is `workflow_dispatch` only.
- `stage*` glob covers all Stage branches without duplication; quoted
  to prevent YAML alias parsing.
- `setup-micromamba@v3` (not v2) per the current official README.
- `micromamba-shell {0}` is the generated wrapper — documented as
  intentional.
- `environment-name: chipseq` explicit alongside the env file.
- `permissions: contents: read` set at workflow level.
- `concurrency` with `cancel-in-progress: true` prevents redundant
  queued runs on active branches.
- No TODOs or TBDs.
