## 3. Design Decisions

### 3.1 chrom_sizes gating: DAG-gated optional output

- **If `chrom_sizes` is empty string:** skip FE/ppois BigWig targets for that genome.
- **If `chrom_sizes` is non-empty but file missing:** existing `validate_samples.py` path validation fails (lines 460-466), no new validation code needed.
- **Users who only want FE/ppois bedGraph are NOT forced to configure `chrom_sizes`.**
- **Docs must not silently promise BigWig unless `chrom_sizes` is configured.**

Rationale: The existing validator already handles "non-empty path → must exist" for all genome resource fields. The only gap is the DAG-level skip when `chrom_sizes` is intentionally empty. This is a feature gating decision, not a validation gap.

### 3.2 Gating helpers location: Snakefile, not validator

Gating helpers are precomputed lists in `workflow/Snakefile`:
- `SIGNAL_BW_SAMPLE_IDS` — treatment samples with `signal_tracks: true` and non-empty `chrom_sizes`
- `SIGNAL_BW_EXPERIMENTS` — multi-biorep experiments whose first treatment sample has `chrom_sizes`
- `_pooled_chrom_sizes(experiment)` — resolves chrom_sizes for an experiment's first treatment sample's genome

No changes to `scripts/validate_samples.py`. The existing `has_genome_resource()` helper is reused.

### 3.3 UCSC environment: new `workflow/envs/ucsc.yml`

Follows the Stage 10e "one tool, one env" pattern. Contains only `ucsc-bedgraphtobigwig` from bioconda. Zero risk of solve conflicts with existing envs.

### 3.4 Sorting: defensive shell sort before conversion

MACS3 `bdgcmp` output is coordinate-sorted in practice but this is not a documented guarantee. `bedGraphToBigWig` requires sorted input. Shell pattern:

```bash
SORTED="$(dirname output)/.tmp_$(basename output .bw).<signal>.sorted.bdg"
trap 'rm -f "$SORTED"' EXIT
LC_ALL=C sort -k1,1 -k2,2n input.bdg > "$SORTED"
bedGraphToBigWig "$SORTED" chrom.sizes output.bw
```

Do NOT use `bedGraphToBigWig stdin` — the binary may not support it.

### 3.5 Pooled experiment genome resolution

Use the first treatment sample's genome for `chrom_sizes` lookup. This matches the existing pattern in `pooled_experiment_qc_summary` (lines 504-507 of `qc.smk`). Mixed-genome pooled experiments are not validated against in this slice. Documented as a known assumption.

### 3.6 README Conda install command

Add the `conda` alternative alongside the existing `micromamba` command. Do NOT replace micromamba. Use the long-form `--file` flag:

```bash
conda env create --file workflow/envs/runner.yml
```

|---|-----------------|-----------------|
| 1 | `chrom_sizes` empty string | No `.FE.bw` / `.ppois.bw` targets in DAG |

## 6. Output Contract: Manifest Field Schema

```tsv
project_id  experiment_id  sample_id  assay  target  genome  output_type  method  path  status  qc_flag
```

Field descriptions:
- `project_id`: user-defined project label (from config)
- `experiment_id`: experiment grouping key
- `sample_id`: sample identifier
- `assay`: `chipseq`, `cuttag`, or `atac`
- `target`: antibody/target name
- `genome`: genome assembly label
- `output_type`: controlled vocabulary (e.g., `final_bam`, `cpm_bigwig`, `macs3_peak`, `macs3_fe_bdg`, `macs3_ppois_bdg`, `macs3_fe_bw`, `macs3_ppois_bw`, `idr_conservative`, `idr_optimal`, `qc_summary`)
- `method`: tool or rule name (e.g., `bowtie2+samtools`, `bamCoverage`, `macs3_callpeak`, `macs3_bdgcmp+bedGraphToBigWig`, `idr`)
- `path`: relative path from `outdir`
- `status`: `present`, `missing`, `not_applicable`
- `qc_flag`: `ok`, `warning`, `fail`, `na`

This schema is a draft for documentation. Implementation of `scripts/make_manifest.py` is deferred to Stage 24a.
